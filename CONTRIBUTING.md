# 贡献指南

我们非常欢迎并感谢您对 AMONEY 数字文明创世协议的贡献！您的参与对于项目的成功至关重要。

在您开始贡献之前，请花一些时间阅读以下指南，以确保贡献过程顺畅高效。

## 行为准则

请务必阅读并遵守我们的 [行为准则](CODE_OF_CONDUCT.md)。我们致力于为所有贡献者和社区成员提供一个开放、包容和尊重的环境。

## 如何贡献

我们欢迎各种形式的贡献，包括但不限于：

*   **代码贡献**：修复 Bug、实现新功能、改进现有代码。
*   **文档改进**：完善白皮书、项目报告、README、注释或任何其他文档。
*   **Bug 报告**：提交详细的 Bug 报告，帮助我们发现和修复问题。
*   **功能建议**：提出新的功能想法或改进建议。
*   **社区支持**：在社区论坛或社交媒体上回答问题，帮助其他用户。

### 提交 Bug 报告

如果您发现了一个 Bug，请通过 GitHub Issues 提交。在提交之前，请先搜索现有 Issue，以避免重复提交。一个好的 Bug 报告应该包含：

*   **清晰的标题**：简明扼要地描述问题。
*   **复现步骤**：详细说明如何重现 Bug。
*   **预期行为**：描述您期望的正确行为。
*   **实际行为**：描述实际发生的问题。
*   **环境信息**：例如操作系统、Node.js 版本、Hardhat 版本等。
*   **截图或日志**：如果适用，提供相关截图或错误日志。

### 提交功能建议

如果您有新的功能想法或改进建议，也请通过 GitHub Issues 提交。一个好的功能建议应该包含：

*   **清晰的标题**：简明扼要地描述建议的功能。
*   **问题描述**：说明当前存在的问题或痛点，以及为什么需要这个新功能。
*   **解决方案建议**：描述您对新功能的设想和实现方式。
*   **潜在影响**：说明新功能可能带来的好处和潜在风险。

### 提交代码贡献

1.  **Fork 仓库**：首先，将本仓库 Fork 到您的 GitHub 账户。
2.  **克隆仓库**：将 Fork 后的仓库克隆到您的本地机器：
    ```bash
    git clone https://github.com/您的用户名/genesis-chain-eth.git
    cd genesis-chain-eth
    ```
3.  **创建新分支**：为您的贡献创建一个新的分支。分支名称应具有描述性，例如 `feature/新功能名称` 或 `bugfix/修复描述`。
    ```bash
    git checkout -b feature/your-feature-name
    ```
4.  **安装依赖**：
    ```bash
    yarn install
    ```
5.  **进行更改**：在您的新分支上进行代码更改或文档修改。
6.  **编写测试**：如果您的更改涉及代码，请确保编写相应的测试，并确保所有测试通过。
7.  **代码风格**：请遵循项目现有的代码风格。
8.  **提交更改**：提交您的更改，并编写清晰的提交信息。请遵循 [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) 规范。
    ```bash
    git commit -m "feat: 添加新功能"
    ```
9.  **推送分支**：将您的新分支推送到您的 Fork 仓库：
    ```bash
    git push origin feature/your-feature-name
    ```
10. **创建 Pull Request (PR)**：在 GitHub 上，从您的 Fork 仓库向本仓库的 `master` 分支创建一个 Pull Request。请确保 PR 描述清晰，并引用相关的 Issue。

## 代码审查

所有 Pull Request 都将经过代码审查。请耐心等待维护者的反馈，并根据反馈进行修改。我们可能会要求您进行一些更改，以确保代码质量和项目一致性。

## 安全披露

如果您发现任何安全漏洞，请不要通过公共 Issue 报告。请参阅我们的 [安全政策](SECURITY.md) 以了解如何负责任地披露漏洞。

再次感谢您的贡献！
